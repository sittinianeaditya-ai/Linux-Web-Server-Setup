Choix Techniques & Théorie

Justification de l'architecture et définitions des concepts clés.

1. Justification des Choix

Système d'Exploitation
Ubuntu 24.04 LTS (64-bit)

Nous avons privilégié une version LTS (Long Term Support) pour garantir la stabilité et la maintenance du poste jusqu'en 2029. L'architecture 64-bit est impérative pour gérer efficacement les applications gourmandes comme WebStorm.

Mémoire Vive (RAM)
4 Go Alloués

C'est un compromis indispensable. Moins de 4 Go rendrait l'utilisation de l'IDE WebStorm pénible. Nous laissons suffisamment de RAM à la machine hôte pour ne pas la ralentir.

Stockage Disque
Allocation Dynamique

Plutôt que de bloquer 80 Go d'un coup, nous utilisons l'allocation dynamique. Le fichier de disque virtuel (.vdi) ne grossit que lorsque nous ajoutons des données, ce qui économise l'espace réel.

Méthode de Déploiement
Déploiement Manuel

Nous avons choisi de transférer et configurer le site manuellement via le terminal (commandes cp, rm, sudo). Ce choix pédagogique renforce notre maîtrise de l'administration système Linux.

2. Concepts Théoriques

C'est une technique permettant de faire tourner plusieurs systèmes d'exploitation (OS) simultanément sur un seul ordinateur physique. Un logiciel "hyperviseur" partage les ressources réelles (CPU, RAM) pour créer des machines isolées.

  • L'émulateur : Il imite le matériel (ex: GameBoy sur PC). Il doit tout "traduire", ce qui est lent.
  • Le simulateur : Il reproduit un comportement pour l'entraînement (ex: simulateur de vol) sans simuler la technique interne.
  • La virtualisation : Elle utilise directement le processeur réel pour exécuter l'OS invité, ce qui est beaucoup plus rapide.

VirtualBox utilise une virtualisation de Type 2 (Hébergée). Le logiciel s'installe comme une application par-dessus le système hôte (Windows/Mac), contrairement au Type 1 (ex: Stratus/ESXi) qui s'installe directement sur le matériel.

Elle est stockée physiquement sur le disque dur de la machine hôte sous forme de fichiers. On trouve généralement un fichier de configuration et un fichier de disque dur virtuel (format .vdi pour VirtualBox).
